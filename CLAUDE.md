## 폴더 구조

```
raw/                     # 원본 자료
  재고현황 내역_*.xlsx   # 현시점 재고 스냅샷 (월별이 아닌 시점 데이터)
  YYYYMM/               # 월별 하위폴더 ex) 202603/
    # --- KSE 제공 (3PL측 정산 자료) ---
    *_詳細_*.xlsx        # KSE 작업 상세 (입고, 출하, 집계 등)
    *_請求書.pdf         # 청구서 (공식 인보이스)
    *_請求内訳確認書.pdf  # 청구내역 확인서 (항목별 상세)
    *銀行情報*.pdf       # 은행 정보 (송금용)
    # --- KATCHERS 제공 (우리측 대조 자료, KSE OMS에서 다운로드) ---
    ORDER_LIST_B2C_*.xlsx       # B2C 오더리스트 (상품 라인별 상세: 상품코드, 수량, 수취인 등)
    ORDER_LIST_B2B_*.xlsx       # B2B 오더리스트 (상품 라인별 상세)
    주문(출고&입고) 내역_*B2C.xlsx  # B2C 주문 내역 (주문 단위 요약: 작업상태, 내부오더번호 등)
    주문(출고&입고) 내역_*B2B.xlsx  # B2B 주문 내역 (주문 단위 요약)

db/                      # 데이터베이스
  logistics.db           # SQLite DB (모든 물류비 데이터)
  init_db.py             # DB 스키마 생성 + 초기 데이터 적재
  loader.py              # 월별 raw 파일 파싱/검증 유틸리티

analysis/                # 분석 리포트 (월별)
  YYYYMM_review.md      # 검토 결과 리포트

history/                 # 히스토리 CSV (백업/참조용)
  monthly_summary.csv   # 월별 요약
  unit_price.csv        # 단가표
  line_items.csv        # 항목별 상세

dashboard.py             # Streamlit 대시보드
```

## 대시보드 실행

```bash
streamlit run dashboard.py
```

대시보드 탭 구성:
1. **월간 요약** - KPI, 비용 구성비 (파이/바 차트)
2. **항목 상세** - 26개 항목별 청구/검증 대조표 (판정 색상 표시)
3. **검토 결과** - 불일치/확인필요 사항 목록, 금액 영향 요약
4. **추이 분석** - 월별 청구 추이, 출하건수/건당비용 (2개월 이상 시)
5. **단가표** - 현행 단가 목록 및 분포 차트

## DB 테이블 구조

- `invoices` - 월별 청구서 (인보이스 단위)
- `line_items` - 항목별 상세 (청구 라인 26개)
- `unit_prices` - 단가 이력 (변동 추적)
- `monthly_metrics` - 월별 운영 지표
- `review_findings` - 검토 결과 (불일치/확인필요)

## 월별 물류비 검토 절차

1. **raw 자료 수집**: `raw/YYYYMM/` 폴더에 원본 파일 배치
2. **수량 대조**: Excel 상세 데이터와 청구내역 확인서의 수량 비교
3. **금액 검증**: 단가 × 수량 = 청구금액 일치 여부 확인
4. **소비세 검증**: 소계 × 10% = 소비세 일치 확인
5. **불일치 기록**: 차이 발견 시 analysis 리포트에 기록
6. **히스토리 갱신**: history CSV 파일에 해당 월 데이터 추가

## 주요 검증 포인트

### KATCHERS vs KSE 대조 (3-way 검증)
- **B2C 주문 건수**: KATCHERS 주문내역 vs KSE 집계(B2C出荷集計) vs 청구서
- **피킹 PCS**: ORDER_LIST 총 수량 vs KSE 집계 PCS vs 청구서
- **송장번호 매칭**: ORDER_LIST 송장 vs KSE 집계 송장 (누락 건 특정)

### KSE 내부 검증
- **배송 사이즈별 건수**: 집계 시트 사이즈별 집계 vs 확인서 배송료 건수
- **오키나와/낙도 중계료**: 개별 주문 중계료 합산 vs 확인서 중계료
- **입고 수량**: 入庫 시트 합계 vs 확인서 입고/재고등록 수량
- **보관료**: PLT 수 별도 확인 필요 (Excel에 월간 보관 데이터 없음)
- **단가 변동**: unit_price.csv 대비 단가 변경 여부
